# Setup And Forward Test Skeleton

This branch intentionally contains only a skeleton. Students should implement
the runtime and then document the real setup commands here.

The expected end-to-end scenario is:

```text
server -> Raspberry Pi daemon -> downstream client/service
```

Use this document to describe:

- how to generate authority, Pi, and server runtime material
- which public keys must be exchanged between machines
- how to run the downstream client
- how to run the Pi daemon
- how to run the server
- which ports must be open
